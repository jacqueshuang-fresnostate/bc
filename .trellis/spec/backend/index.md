# 后端开发规范

> Rust 后端的初始项目约定。

---

## 概览

后端是彩票软件管理系统的 Rust HTTP API。首期实现使用 Axum、Tokio、Serde 和 tracing。代码按技术层组织，领域模型放在 `domain/`，HTTP 路由放在 `routes/`。

本项目从文档起步，因此这些规范是当前初始事实来源。真实约定变化时，需要同步更新本目录文档。

## 核心规则

- 优先使用小模块和明确的公开导出。
- 领域类型需要独立于 Axum 的请求/响应提取器。
- 公开接口必须返回统一 API 响应信封。
- 占位和演示数据必须清晰隔离，直到接入数据库持久化。
- 彩票资金流转、派奖、机器人自动执行等逻辑没有测试时不能上线为真实能力。
