# 日志规范

> 本项目后端日志记录方式。

---

## 概览

后端使用 `tracing` 和 `tracing-subscriber`。日志需要帮助排查用户、财务、彩票、开奖、返利和机器人行为，同时不能暴露敏感数据。

---

## 日志级别

- `debug`：本地开发细节、派生计算和非敏感诊断信息。
- `info`：服务启动、路由组初始化、定期开奖执行、机器人任务开始和结束。
- `warn`：可恢复的异常状态、跳过任务、可疑但非致命行为。
- `error`：请求失败、财务变更失败、开奖失败、外部 API 失败。

---

## 结构化日志

优先使用结构化字段，而不是很长的格式化字符串：

```rust
tracing::info!(lottery_id = %lottery_id, issue = %issue, "draw completed");
```

相关场景中应包含 `user_id`、`lottery_id`、`order_id`、`issue` 等标识。

---

## 应记录内容

- 服务启动和绑定地址。
- 外部开奖 API 失败与重试。
- 会改变余额的操作。
- 订单生命周期变化。
- 机器人创建的合买计划或订单活动。
- 管理员权限敏感操作。

---

## 禁止记录内容

- 密码、密码哈希、令牌、会话 ID。
- 完整邮箱地址，除非客服诊断必须使用。
- 支付密钥或供应商凭据。
- 可能包含个人或财务数据的原始请求体。
