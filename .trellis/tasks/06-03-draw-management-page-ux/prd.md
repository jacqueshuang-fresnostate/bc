# 优化开奖期号与开奖源页面

## 背景

“开奖期号与开奖源”页面已承载期号列表、创建期号、计划预览、开奖执行、开奖源配置、自动任务、常驻调度配置和历史。当前这些模块全部平铺在同一长页面内，导致主列表扫描困难、右侧表单过多、运营用户无法快速判断当前重点操作。

## 目标

1. 页面首屏保留关键概览：期号总数、待开奖/已开奖、API 来源数量、调度状态。
2. 主工作区用分段标签拆分为：
   - 期号管理
   - 开奖源配置
   - 自动任务与调度
3. 主页面优先展示列表、摘要、状态和操作入口，不长期展示大表单。
4. 创建期号、执行开奖、开奖源新建/编辑、调度配置使用 Semi UI `SideSheet` 打开。
5. 保留现有 API、hook 和业务能力，不改变后端契约。
6. 页面仍保持紧凑后台风格，避免营销式布局和嵌套装饰卡片。

## 范围

- 修改 `admin/src/pages/DrawManagementPage.tsx`。
- 更新 `架构设计.md`、`TODO.md` 和前端规范相关任务记录。
- 不新增后端接口。

## 验收标准

- `npm run build` 通过。
- 期号列表首屏不再被创建/开奖/源配置表单挤压。
- 点击“创建期号”打开 SideSheet，可创建手动期号、按计划生成、预览和批量生成。
- 点击期号行的“开奖”打开 SideSheet，保留封盘、开奖、取消能力。
- 点击开奖源“新建/编辑”打开 SideSheet，保留保存、删除、复用彩种选择。
- 自动任务和调度分到独立标签页，调度配置通过 SideSheet 打开。
