# 修复福彩 3D 真实期号生成

## 背景

当前开奖期号生成服务对所有彩种统一使用开奖时间格式 `YYYYMMDDHHMMSS` 作为期号。例如福彩 3D 在 2026-06-03 生成的期号会是 `20260603210015`。但 API68 福彩 3D 接口返回的真实期号是 `preDrawIssue`，例如当前最新为 `2026143`，下一期应为 `2026144`。错误的内部时间戳期号会导致后续 API 开奖无法按 `preDrawIssue` 命中。

## 目标

1. 福彩 3D 和复用同一 API68 来源的排列 3，在自动生成、预览和批量生成期号时，优先使用 API68 最新 `preDrawIssue` 作为真实期号基线。
2. 当 API68 最新期号为 `2026143` 时，2026-06-03 的下一期生成结果应为 `2026144`。
3. 如果本地已经存在同彩种正常期号，例如 `2026144`，继续生成时应跳过并生成 `2026145`。
4. 没有配置 API 来源的彩种继续沿用当前按开奖时间生成期号的逻辑。
5. 如果已配置 API 来源但外部源无法提供最新期号，后台生成/预览接口不能生成错误的时间戳期号，应返回清晰错误；常驻调度补期时应跳过该彩种并记录原因，避免整轮调度失败。

## 范围

- 后端 `ApiDrawSourceRepository` 增加最新外部期号解析能力。
- 后端 `draw_generation` 在计划期号时识别 API 来源最新期号，生成真实递增期号。
- 后端常驻调度对单个彩种补期失败做跳过记录。
- 更新 API 契约、架构文档和 TODO。

## 验收标准

- API68 静态样本最新 `preDrawIssue=2026143` 时，`fc3d` 生成下一期为 `2026144`。
- 同一来源复用的 `pl3` 也生成 `2026144`。
- 已存在 `fc3d/2026144` 时，再生成得到 `2026145`。
- 无 API 来源的每日开奖彩种仍按时间戳期号生成。
- `cargo fmt --check`、`cargo check`、`cargo test` 通过。
