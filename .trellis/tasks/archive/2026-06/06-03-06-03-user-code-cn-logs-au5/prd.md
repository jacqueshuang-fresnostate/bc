# 全员邀请码中文日志与澳洲 5 分彩接入

## 背景

当前用户管理显示的邀请码来自邀请关系聚合，只覆盖已有邀请关系的代理用户。新的业务要求是每个用户都有自己的邀请码，但只有代理用户的邀请码具备邀请功能；普通用户的邀请码仍可展示，使用时需要提示邀请码无效。同时后端日志输出需要始终使用中文 message，新增澳洲 5 分彩并接入 API68 开奖源。

## 目标

1. 每个用户摘要都有单个 `inviteCode` 字段。
2. 代理用户的邀请码可用于创建邀请关系；普通用户的邀请码创建邀请关系时返回“邀请码无效”。
3. 用户管理页面展示单个邀请码，并标识代理码有效、普通用户码无效。
4. 后端 `tracing`/`panic` 等运行日志 message 改为中文。
5. 新增澳洲 5 分彩，号码类型为 5 位数字，使用 API68 `CQShiCai/getBaseCQShiCaiList.do?lotCode=10010`。
6. 同步更新架构设计、TODO 和 API 契约。

## 非目标

1. 不新增用户端注册页面。
2. 不新增二维码、邀请链接或真实邀请码生成服务。
3. 不接入数据库持久化。

## 验收

1. 用户列表中每个种子用户都有 `inviteCode`。
2. 用普通用户邀请码创建邀请关系返回业务错误“邀请码无效”。
3. API 开奖源列表包含澳洲 5 分彩来源，并绑定 `au5`。
4. `cargo fmt --check`、`cargo check`、`cargo test`、`npm run build` 通过。
