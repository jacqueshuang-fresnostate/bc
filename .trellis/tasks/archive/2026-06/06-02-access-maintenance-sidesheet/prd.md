# 用户权限维护侧边栏

## 背景

用户要求用户管理的“用户维护”、管理员管理的“账号维护”、角色权限的“角色维护”不要直接常驻显示在页面上，而是通过 SideSheet 打开。当前页面采用列表与维护表单并排布局，表单始终暴露，占用运营列表空间。

## 目标

- 用户管理主界面只直接展示用户列表和操作入口。
- 管理员管理主界面只直接展示后台账号列表和操作入口。
- 角色权限主界面只直接展示角色列表和操作入口。
- 新建和编辑入口打开 Semi UI SideSheet，表单内容在抽屉内完成。
- 保存或删除后保持现有数据刷新链路，并关闭对应 SideSheet。

## 非目标

- 不修改后端接口和数据库结构。
- 不改变系统设置、注册配置的现有布局。
- 不新增权限校验逻辑。

## 验收标准

- “用户维护”“账号维护”“角色维护”不再作为页面常驻卡片显示。
- 点击新建用户、编辑用户能打开用户维护 SideSheet。
- 点击新建账号、编辑账号能打开账号维护 SideSheet。
- 点击新建角色、编辑角色能打开角色维护 SideSheet。
- `npm run build` 通过。
- `架构设计.md` 和 `TODO.md` 记录本次调整。
