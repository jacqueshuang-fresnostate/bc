# 在线客服移除后台新建会话

## 背景

用户要求在线客服不需要后台新建会话，只需要回复用户发过来的信息。当前“在线客服”页面左侧包含“新建会话”表单，允许管理员主动创建客服会话，不符合新的运营定位。

## 目标

- 移除“在线客服”页面中的“新建会话”卡片和创建会话逻辑。
- 页面只展示用户已发起的会话列表、会话详情、消息记录、状态/优先级/客服分配和后台回复入口。
- 保留现有 Semi UI `Chat` 消息展示。
- 不修改后端接口和内存仓储，避免影响后续用户端发起会话能力。

## 非目标

- 不实现用户端客服入口、WebSocket 或实时消息。
- 不删除后端创建会话 API。
- 不重构客服 hook 或 API client。

## 验收标准

- 在线客服页面不再显示“新建会话”“创建会话”“会话 ID”“首条消息”等后台创建入口。
- 选中已有会话后仍能查看 `Chat` 消息记录。
- 后台回复表单仍可发送回复。
- `npm run build` 通过。
- `架构设计.md` 和 `TODO.md` 记录本次调整。
