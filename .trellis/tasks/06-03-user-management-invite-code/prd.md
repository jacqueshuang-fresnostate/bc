# 用户管理显示邀请码

## 背景

用户管理页面当前只展示用户基础资料、余额、上级代理和状态，运营无法在用户列表中直接看到代理用户已绑定的邀请码。邀请关系管理中已经维护 `inviteCode`，但用户管理页只具备 `users` 权限，不能额外依赖需要 `rebates` 权限的邀请管理接口。

## 目标

1. `/api/admin/users`、用户详情、用户创建、更新和状态变更响应中增加只读 `inviteCodes` 字段。
2. `inviteCodes` 从邀请关系按邀请人聚合得出，不允许用户维护表单直接编辑。
3. 用户管理表格新增“邀请码”列，展示用户作为邀请人拥有的邀请码，无邀请码时显示未配置。
4. 同步更新架构设计、API 契约和 TODO 记录。

## 非目标

1. 不新增邀请码生成服务。
2. 不调整邀请关系创建、审核、返利发放逻辑。
3. 不把用户管理页改为依赖邀请管理接口。

## 验收

1. 后端测试覆盖邀请码按邀请人聚合。
2. `cargo fmt --check`、`cargo check`、`cargo test` 和 `npm run build` 通过。
3. 前端用户管理列表能看到 `agent_alpha` 的 `AGENT10001`、`AGENT10002`。
