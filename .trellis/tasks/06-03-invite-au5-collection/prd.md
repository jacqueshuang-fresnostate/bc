# 邀请码与澳洲 5 分彩采集修正

## 目标

修正用户邀请码、后台中文日志和澳洲 5 分彩采集配置，让后台行为符合运营预期：每个用户都有邀请码，但只有代理邀请码可用于邀请；后台日志始终使用中文业务描述；澳洲 5 分彩使用指定 API 采集开奖数据。

## 已知需求

1. 每个用户都有邀请码。
2. 只有代理用户的邀请码具备邀请功能。
3. 普通用户的邀请码被用于邀请时，需要提示“邀请码无效”。
4. 后台打印日志需要始终为中文。
5. 澳洲 5 分彩采集接口为 `https://api.api68.com/CQShiCai/getBaseCQShiCaiList.do?lotCode=10010`。

## 验收标准

1. 用户列表和用户详情中普通用户、代理用户都能看到邀请码。
2. 代理邀请码可以创建邀请关系；普通用户邀请码用于邀请时返回中文错误“邀请码无效”。
3. 澳洲 5 分彩的 API 开奖源使用 `CQShiCai/getBaseCQShiCaiList.do?lotCode=10010`。
4. 后端新增或修改的日志内容为中文，不新增英文业务日志。
5. 相关单元测试覆盖邀请码有效性和澳洲 5 分彩采集接口配置。
6. `架构设计.md`、`TODO.md` 根据功能变更同步中文记录。

## 非目标

1. 不在本阶段把邀请、用户或开奖源仓储迁移到 PostgreSQL。
2. 不修改第三方 API 返回结构之外的采集协议。
3. 不改变已有 API 响应信封结构。

## 技术备注

1. 当前工作区存在用户未提交改动：`admin/vite.config.ts`、`backend/src/main.rs`、`.idea/`，本任务不得提交这些文件。
2. 需要优先检查 `services::access`、`services::invite`、`services::draw_api`、用户管理前端页面和日志规范。
3. 代码中已经存在 `au5` 彩种、`api68-au5` 默认来源和普通用户邀请码无效的后端测试，本轮重点补齐前端表单不会覆盖邀请码、邀请关系页面自动使用邀请人邀请码，以及日志打印时不输出英文错误描述。
